# spec-sonar 輸出目錄規範

<!-- spec-sonar v1.0 -->

## 目錄結構

```
{project}/
├── spec/                          ← 唯一真相源（single source of truth）
│   ├── project-spec.md            ← 人讀完整規格
│   ├── STATE_FINAL.json           ← 收斂狀態存檔（provenance）
│   ├── goal-graph.json            ← 機讀依賴圖
│   ├── contracts/
│   │   └── C{n}-{slug}.md         ← 凍結介面契約
│   └── goals/
│       └── G{n}{phase}.md         ← 自包含 goal 檔
├── adapters/                      ← spec/ 的投影，永不手改，只重新生成
│   ├── CLAUDE.md
│   ├── .cursor/rules/*.mdc
│   ├── copilot-instructions.md
│   └── system-prompt.md
├── audit/                         ← Audit Mode 專屬
│   ├── dark-zone-findings.md
│   ├── dependency-conflicts.md
│   └── remediation-goals.md
└── scan/
    └── project-fingerprint.md     ← project-scanner.py 產出
```

## 全域規則

1. **Provenance 註記**：每個生成檔頭部必有
   `<!-- generated by spec-sonar {tool} v{x} from {source} @ {ISO time} — do not hand-edit -->`
2. **投影不手改**：adapters/ 是投影——改規格必須改 spec/ 再重新生成，
   防止四平台漂移。
3. **部署映射**：
   - `adapters/CLAUDE.md → 專案根目錄`
   - `adapters/.cursor/rules/ → .cursor/rules/`
   - `adapters/copilot-instructions.md → .github/`
   - `adapters/system-prompt.md → 貼入通用 agent 的 system message`

## 各檔案必要欄位

| 檔案 | 必要欄位 | 格式規則 |
|------|----------|----------|
| project-spec.md | 專案目標（一句話可驗收）、目標用戶、功能模組+驗收條件、明確不做、技術棧、已知風險 | output-templates.md 的 README+CLAUDE 合併源 |
| STATE_FINAL.json | schema 1.1 全欄位（含 tech_stack、known_risks、revision_log、pending_conflict） | 見 output-templates.md 檔案三 |
| goal-graph.json | 見 schemas/goal-graph.schema.json | 必過 schema 驗證 |
| contracts/C*.md | 契約 ID、版本、凍結宣告、字面介面定義（JSON 實例/數值表）、消費者清單 | 範例：`### 事件 unit:placed` + 完整 payload JSON |
| goals/G*.md | 標頭（id/tier/depends/blocks）、五元素、前置裁決、凍結介面複本、明確不做複本、BLOCKED 協議、自檢 footer | 完整範例見 examples/mathdefense/goals/G4a.md |
| audit/ 三檔 | 見 modes/audit-mode.md | — |
| project-fingerprint.md | 8 節固定結構 | < 20KB |

## contracts/C*.md 範例片段

```markdown
# C1 — Socket 事件與 payload 凍結表
- version: 1.0
- 凍結宣告：goal 執行期間不可修改。要改 → 回 decomposer 發 C1.v2
- consumers: G1, G2, G3, G4a, G4b, G5, G6

### 事件 unit:placed（S→C）
{"x":3,"y":2,"unit":{"type":"peashooter","team":"plant","hp":100,"attackPower":10,"range":5}}
```
