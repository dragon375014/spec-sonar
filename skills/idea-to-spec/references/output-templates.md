## 檔案三：STATE_FINAL.json（schema 1.1）

直接輸出最終 `<STATE>` 區塊內的 JSON 內容，加上以下欄位：

```json
{
  "schema_version": "1.1",
  "generated_at": "{{ ISO 時間戳 }}",
  "round": 7,
  "lang": "{{ 語言 }}",
  "user_tech_level": "{{ 等級，複合情境允許 {基準值}+partner }}",
  "bright": [
    { "id": "B1", "claim": "{{ 確立項目 }}", "round": 1, "depends_on": [] }
  ],
  "dark": [ "{{ 剩餘未解決項目，應 <= 2 }}" ],
  "complexity": "{{ mvp / full / enterprise }}",
  "duplicate_check": "{{ done-unique / done-similar }}",
  "infeasible_flag": false,
  "pending_conflict": null,
  "revision_log": [
    { "round": 5, "action": "retract | conflict-resolved | late-add", "id": "B3", "old": "…", "new": "…" }
  ],
  "tech_stack": {
    "frontend": "{{ 框架 }}",
    "backend": "{{ 框架 / BaaS }}",
    "database": "{{ 資料庫 }}",
    "deployment": "{{ 平台 }}",
    "integrations": [ "{{ 清單，可空 }}" ]
  },
  "known_risks": [ "{{ 收斂過程發現的風險 }}" ],
  "next_focus": null,
  "resume_instruction": "把這份 JSON 貼到新 session 的第一則訊息，系統會從這裡繼續。"
}
```

欄位異動說明（v1.0 → v1.1）：
- `bright` 由字串陣列升級為物件陣列（id / claim / round / depends_on）
- 正式收編 `tech_stack`、`known_risks`（v1.0 時代是 ad-hoc 欄位）
- 新增 `revision_log`、`pending_conflict`（衝突處理協議的存檔）
- `user_tech_level` 允許 `{基準值}+partner` 複合值
